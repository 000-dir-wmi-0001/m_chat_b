## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## Frontend Integration Guide: Temporary One-to-One Chat

### Health Check (REST)
- **Endpoint:** `GET /health`
- **Response:** `{ status: 'ok', timestamp: <number> }`

---

### WebSocket Events

#### 1. `createRoom`
- **Client → Server:**  `{}`
- **Server → Client:**  `{ code: '123456' }`
  - Generates a new 6-digit room code and creates a room.
  - The client is joined to the room.

#### 2. `joinRoom`
- **Client → Server:**  `{ code: '123456' }`
- **Server → Client:**
  - Success:  `{ success: true, messages: [ { sender, text, timestamp } ] }`
  - Failure:  `{ error: 'Room not found' }` or `{ error: 'Room full' }`
  - The client is joined to the room if possible.

#### 3. `sendMessage`
- **Client → Server:**  `{ code: '123456', text: 'Hello!' }`
- **Server → Client:**  `{ success: true }`
  - Broadcasts to room: `newMessage` event: `{ sender, text, timestamp }`

#### 4. `userJoined`
- **Server → Room:**  `{ userId: <socket.id> }`
  - Emitted when the second user joins the room.

#### 5. `userLeft`
- **Server → Room:**  `{ userId: <socket.id> }`
  - Emitted when a user disconnects.

#### 6. `roomExpired`
- **Server → Room:**  `{}`
  - Emitted when the room expires (after 20 minutes or all users leave).

---

### Notes
- All rooms and messages are stored in memory only.
- Each room holds exactly 2 users.
- No authentication, no persistent storage.
- Disconnecting, refreshing, or server restart destroys the room and messages.

**Frontend should connect to the backend WebSocket (default: `ws://localhost:3000`) using socket.io-client.**
